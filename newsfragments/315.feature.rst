set_code_transaction for EIP-7702

### Overview
This feature introduces a new type of transaction, set_code_transaction, which enables the usage of EIP-7702. This transaction type allows users to set the code of a contract without deploying a new contract. This feature is useful for updating the code of a contract without changing its address or state.

### usage
You need to get signed one or more signed authorisations from an EOA willing to have a smart contract code associated with the EOA, this the essence of EIP-7702
an autorisation is of this form:
```json
{'chainId': 7072151312,
 'address': "0x3e6C95D880401e4E36Ee62f4eBDed346e1aDf42d",
 'nonce': 2,
 'yParity': 1,
 'signerR': 22595136657293516951860802422974352017713294017347016159649668416801694741909,
 'signerS': 23624588567578401597292901415360791985725988995245670164012927046435484403948}
```
where:
- address is the address of the smart contract code to be associated with the EOA
- nonce is the nonce of the EOA, it is used to prevent replay attacks
the rest of the fields are the signature of the first 3 fields by the EOA

Create the transaction as before, adding a new field named autorisationList, for instance:

transaction_dict = {'authorisationList':[my_auth1, my_auth2], "to": some_address}


### signature of the authorisation, example code:
```python
from eth_account import Account
from eth_utils import decode_hex
from eth_keys import keys
from eth_rlp import SignedAutorizationRLP, AuthorizationRLP

#
# you need to have the EOA private key, the EOA address and the code address
# in variables named signer_EOA_private_key, signer_EOA_address, code_address
# and an instanciated web3 object named w3
#
chain_id = w3.eth.chain_id
signer_nonce = w3.eth.get_transaction_count(signer_EOA_address)
my_authorization = AuthorizationRLP(chain_id, bytes.fromhex(code_address[2:]) ,signer_nonce)
authority_keys = keys.PrivateKey(decode_hex(signer_EOA_private_key))
[v, r, s] = authority_keys.sign_msg_hash(my_authorization.hash()).vrs 
my_authorization_signed_dict = my_authorization.as_dict().update({"yParity":v,"signerR":r,"signerS":s})
my_authorization_signed = SignedAutorizationRLP(chain_id, bytes.fromhex(code_address[2:]),signer_nonce,v,r,s)
my_authorization_signed.as_dict() # this is the signed authorization to be added to the transaction_dict
```
